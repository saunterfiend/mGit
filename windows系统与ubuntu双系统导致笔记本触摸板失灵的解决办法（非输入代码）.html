windows系统与ubuntu双系统导致笔记本触摸板失灵的解决办法（非输入代码）

作者: admin 分类: Ubuntu,网络应用 发布时间: 2016-07-17 23:52

先说一下我现在的笔记本使用的系统，windows10 64位＋ubuntu14.04 64位。

这几天把ubuntu装好后本来毫无问题的，用的飞起～可是昨天不知道什么原因，触摸板突然失灵了，插上鼠标可以用，就是触摸板不能用。

同志们，重点来了，如果网上说的那些输入代码的方法都不可行的话，切换到windows系统下，检查一下windows系统下你的触摸板是不是已经被禁用了，或者是不是设置插入鼠标后禁用触摸板了，启动它。你可要知道，windows系统和ubuntu系统对于触摸板的设置可是通用的。

———-下面这些看不看都可以，只是叙述经过————

 

对此事件做一下记录：

 

首先我在网上找到了一些解决方法：

 

 

在终端输入以下代码：

sudo modprobe -r psmouse

 

sudo modprobe psmouse

 

重新加载相应模块就可以了。

 

 

 

还有的说： 在终端输入以下代码：

sudo modprobe -r psmouse

 

sudo modprobe psmouse proto=imps

 

然后再输入

 

sudo gedit /etc/modprobe.d/options

 

再打开的页面中添加：

 

options psmouse proto=imps

 

保存后重启。

 

以上方法我试过之后都不行，于是接着找，最后在一篇文章中得到启发：

 

文章中有这么一段话：“会不会是windows下因为插入鼠标禁用了touchpad，而导致ubuntu下也被禁用了。然后就去windows下，拔去鼠标，触摸板恢复使用，再重启进入ubuntu。结果真的可以了！”

 

虽然我不是这个原因，但是却让我找到了真正的原因，原来是我在windows系统下禁用了触摸板，功能键＋F10，在windows系统下开启触摸板，再切回到ubuntu系统，问题解决，哈哈～
